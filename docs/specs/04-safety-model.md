# Spec 04 — 安全模型

> 清理软件的生命线是"从不误删"。本 spec 定义删除路径上的全部防线。

## 1. 分层防线总览

```
保护清单(一票否决) → 风险分级(02) → 用户确认 → 占用/前置检查
   → 事务化执行 → 隔离区/回收站 → 撤销中心 → 审计日志
```

## 2. 保护清单（Protected Paths，硬编码于引擎，不可被规则包覆盖）

- `%WINDIR%` 核心（除白名单化的缓存子目录）、`Program Files*` 可执行本体、驱动库、EFI/恢复分区
- 用户资产根：Documents/Pictures/Videos/Desktop（及其重定向真实位置）——只允许 T3 浏览，不允许任何规则自动推荐
- 浏览器书签/密码/Cookies 库文件、邮件存储（PST/OST）、证书库
- 任何带 `FILE_ATTRIBUTE_REPARSE_POINT` 的链接本体（防止穿越删除真实目标）
- 卷根、`System Volume Information`、`$Recycle.Bin` 结构本体（清空回收站走 Shell API）
- 路径规范化后再校验（防 `..`、符号链接、8.3 短名绕过）

## 3. 执行引擎

### 3.1 事务化批次
- 每次清理是一个**批次（batch）**：先生成不可变执行计划（文件列表 + 动作 + 预估空间），用户确认的是这个计划的快照；执行时逐项校验文件未被替换（路径 + 大小 + 修改时间指纹匹配才删）
- 执行中系统休眠/崩溃可恢复：批次日志先写后删（WAL 思路）

### 3.2 删除通道（按项声明，02/03 规则指定）
| 通道 | 用途 |
|------|------|
| 回收站（Shell API） | 小体量、用户可见性高的项 |
| **自建隔离区** | 默认通道：移动（同卷 rename，秒级）到 `<卷>\CCZen.Quarantine\<batch-id>\`，保留原路径映射清单 | 
| 直接删除 | 仅 T0 中体量巨大且语义绝对安全的项（如 WU 缓存），且用户在设置中显式允许 |
| 系统 API 委托 | 休眠文件（powercfg）、WinSxS（DISM）、存储感知联动——绝不自己动这些文件 |
| 停服务包裹 | 需要先停服务的项（wuauserv 等）：停→删→恢复，失败则恢复服务并中止 |

### 3.3 占用与锁
- 删除前 Restart Manager 会话查询占用进程；被占用项默认跳过并在结果中说明（提供"退出应用后重试"引导）
- 绝不强制解锁/杀进程（除非用户对特定应用点了"退出并清理"，且只用应用自身的优雅退出方式）

## 4. 隔离区与撤销

- 隔离区默认保留 **7 天**（可调 1–30 天），到期后台静默真删；空间紧张时提示用户但仍需确认
- **撤销中心**：按批次一键还原（同卷 rename 回原路径；原路径已有新文件则还原为 `.restored` 副本并提示）
- 隔离区计入"CCZen 可再释放空间"，UI 全程可见，杜绝"清理了空间没变"的困惑
- 隔离区目录 ACL 限制为管理员 + 当前用户

## 5. 规则包与更新安全

- 规则包/Adapter 更新文件必须 **Ed25519 签名**，公钥固化在签名的主程序内；校验失败拒绝加载并回退内置基线包
- 主程序与提权服务均 Authenticode 签名；自动更新走 HTTPS + 包签名双校验
- 规则包无代码执行能力（纯声明式数据），枚举器/动作只能引用引擎内置的有限动作集

## 6. 权限模型

- UI 进程始终标准权限；需要 MFT 读取或系统区清理时，按需 UAC 提权启动**助手进程**（或安装可选服务），最小接口 IPC（命名管道 + 调用方签名校验）
- 助手进程只接受白名单动作（枚举卷、执行已签名批次计划），不提供任意路径删除接口

## 7. 审计与可观测

- 本地审计日志：每批次的计划、实际执行、跳过原因、还原记录（JSON Lines，滚动上限）
- 无任何默认联网行为；更新检查可关闭

## 8. 测试要点

- 保护清单穿透测试：符号链接/junction 指向 Windows 目录的 cache 命名目录 → 必须拒绝
- 断电/杀进程注入测试：批次执行到一半 kill → 重启后无半删状态、可继续或回滚
- 还原冲突、跨卷隔离区禁用（隔离区必须同卷，跨卷项自动改走回收站）
- 权限探针：标准用户调用助手进程执行未签名计划 → 拒绝
